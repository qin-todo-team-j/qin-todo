## Git Branch

`main`

- 本番環境

`develop`

- ステージング環境
- main と develop はなるべく同じ状態にする。

`feature/[issue番号/issue名]`

- 開発の際は基本的に develop を最新にしてから feature ブランチに分岐し、PR を作成後 develop にマージする。
